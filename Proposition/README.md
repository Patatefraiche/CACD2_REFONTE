# CACD2 — Proposition direction artistique de transition

**Livraison juin 2026 · v0.1**

Ce dossier contient une proposition de direction artistique de transition pour CACD2, conçue pour vivre dans Claude (skill) autant que dans le navigateur (HTML/CSS). Elle sert de rampe entre la situation actuelle (production "bancale" via Claude/Stitch/PowerPoint sans direction unifiée) et la refonte d'identité post-fusion Data AI / Data Lab.

## Ouvrir la démo en 30 secondes

```bash
open index.html
```

Ou double-clique sur `index.html` dans le Finder. Aucune dépendance, aucun build : ça marche dans n'importe quel navigateur moderne (Safari, Chrome, Firefox).

Pour tester le **dark mode** : clic sur l'icône lune en haut à droite. Pour exporter en PDF (1 page = 1 slide A4 paysage) : `Cmd + P` → "Enregistrer en PDF" en mode paysage.

## La direction en 30 secondes

| **Palette** | Mint #E8F5EC (fond signature) · Vert CACD2 #01A04C · Vert sapin #0E2E22 · Teal #2EB5A8 |
| **Display** | Poppins Bold (conservé du DS existant) |

| **Layout** | Tuiles modulaires, sidebar éditoriale verticale, chiffres outline géants |
| **Visuels** | 3D matériel pour identité/hero, line-art vert pour le fonctionnel |

Voir `DIRECTION.md` pour le détail (palette dark mode, règles d'usage, etc.).

## Anti-clichés stricts

Bannis dans toute production CACD2 :
- Fusées, nuages IA, cerveaux, robots, ampoules
- Em-dashes (—) gratuits dans le copy
- Point vert "● Generated by Claude"
- Gradients pastels purple/pink en blob (signature Claude default)
- Fond noir comme primaire (exception : citation, transition)
- Stock imagery, Storyset, undraw

Liste complète dans `skill-claude/content-rules.md`.

## Comment utiliser le skill Claude

### Option A — Skill local Claude Code

1. Copier le dossier `skill-claude/` dans `~/.claude/skills/cacd2-design/`
2. Dans une nouvelle conversation Claude Code, le skill apparaît automatiquement dès qu'on parle de générer du contenu CACD2
3. Tester : "Génère-moi une slide CACD2 type stats sur 70 collaborateurs, 45 projets, 6 ans"

### Option B — En conversation directe

1. Charger en contexte les fichiers : `DIRECTION.md`, `skill-claude/SKILL.md`, `skill-claude/content-rules.md`, et le template adapté
2. Demander : "Utilise le DS CACD2 pour générer [artefact]"
3. Claude produit du HTML conforme aux tokens et règles

### Workflow type

```
User → "Slide titre chapitre 03 : 'Notre approche'"
  ↓
Claude lit slide-title.html
  ↓
Remplace "04" par "03", "Expertises" par "Notre approche"
  ↓
Sidebar label = "APPROCHE", page = "03 / N"
  ↓
Output : HTML self-contained, prêt à présenter
```

## Notes sur les choix

**Pourquoi vanilla HTML/CSS et pas React/Tailwind ?**
Un skill Claude doit pouvoir générer des artefacts sans dépendance externe. Le HTML monobloc est lisible, auditable, fonctionne en local sans build, et s'exporte en PDF via le navigateur. Si la version site final passe par un framework, les tokens et règles restent transposables.

**Pourquoi Fraunces Italic en accent ?**
C'est le détail signature qui permet de sortir du look "Claude default" sans casser le DS existant. Un seul mot italique dans un titre Poppins crée instantanément un caractère éditorial premium (référence : Magnetism Agency). À utiliser avec parcimonie — la règle d'or est "un seul mot par titre".

**Pourquoi conserver la sidebar éditoriale ?**
C'est le marqueur le plus distinctif des slides actuelles de Charlotte (sidebar verticale + page number 08/14). Le garder assure la continuité visuelle avec les présentations déjà produites, et c'est un détail rare en B2B tech français.
