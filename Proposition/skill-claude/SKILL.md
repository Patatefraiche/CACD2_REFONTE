# CACD2 — Skill de génération d'artefacts

Tu génères des artefacts visuels CACD2 en respectant la direction artistique de transition. CACD2 est filiale 100% Crédit Agricole S.A., positionnée comme "Fabricants et opérateurs de solutions IT" pour le groupe.

## Avant de générer — charger en contexte

Quand tu démarres, lis dans cet ordre :
1. `../DIRECTION.md` — palette, typo, layout, ton (référence courte)
2. `content-rules.md` — règles éditoriales détaillées et anti-clichés
3. `../tokens.css` — variables CSS exactes
4. Le template adapté dans `templates/`

## Workflow

### Étape 1 — Comprendre la demande

Identifier :
- **Type** : slide unique, deck multi-slides, fiche produit, propal, article, bannière
- **Format** : 16:9 paysage (slides), A4 portrait (docs imprimés), mobile (bannières)
- **Audience** : interne CACD2, client groupe Crédit Agricole, prospect externe
- **Volume** : nombre de slides/sections

Si **ambigu**, poser UNE question maximum. Sinon, faire un choix raisonnable et l'expliciter brièvement.

### Étape 2 — Choisir le bon template

| Besoin | Template | Pour quoi |
|---|---|---|
| Chapitre / transition | `slide-title.html` | Gros chiffre outline + label, fond vert sapin |
| Section editoriale | `slide-section.html` | Eyebrow + titre + sous-titre, fond mint |
| Chiffres clés | `slide-stats.html` | 3-6 stats en grille |
| Contenu visuel + texte | `slide-content.html` | 2-col mockup/photo gauche, titre + texte droite |
| Citation forte | `slide-quote.html` | Citation pleine page, fond mint |

### Étape 3 — Adapter le contenu

- Garde la structure HTML du template
- Remplace UNIQUEMENT les zones marquées `<!-- @cacd2:content -->`
- Garde les classes CSS et les liens vers `tokens.css` / `styles.css`
- Page indicator : compteur cohérent (`01 / N`, `02 / N`, ...)
- Sidebar label : nom de la section en uppercase court
- Mets **un seul mot en Fraunces italic** par titre quand c'est naturel (`<em>mot</em>`)
- Vocabulaire CACD2 (voir `content-rules.md`)

### Étape 4 — Check-list anti-clichés

Avant de livrer, vérifier :

- [ ] Pas d'em-dash (`—`) injecté — utiliser un tiret simple si besoin
- [ ] Pas de fusée, nuage IA, cerveau, robot, ampoule
- [ ] Pas de point vert "● Generated by Claude"
- [ ] Pas de gradient pastel pourpre/rose/bleu en blob
- [ ] Pas de fond noir comme primaire (exception : citation, transition courte)
- [ ] Pas de stock imagery (Storyset, undraw, illustrations IA génériques)
- [ ] Pas de formule creuse ("transform with AI", "next-gen", "disrupt")
- [ ] Pas d'emoji dans les titres
- [ ] Iridescent gradient (orange/pink/purple/blue) **uniquement** sur le hero 3D
- [ ] Vocabulaire CACD2 respecté (Digital Factory, IT as a Service, etc.)

### Étape 5 — Livrer

- HTML complet, prêt à ouvrir dans un navigateur
- Lien vers `tokens.css` et `styles.css` (en chemin relatif depuis l'emplacement de sortie)
- Si le user veut un fichier portable (sans dépendance externe), inline les CSS critiques dans `<style>` au lieu des `<link>`
- Mentionner brièvement le template utilisé et les choix éditoriaux (1 ligne max)

## Couleurs et typo — raccourci

```
--brand-green        #01A04C   /* logo CACD2 */
--brand-green-deep   #0E2E22   /* titres puissants, fonds transition */
--brand-green-vivid  #2EE07A   /* accent dark mode */
--surface-mint       #E8F5EC   /* fond signature */
--surface-base       #FFFFFF
--ink-strong         #0A0F0D

--font-display       Poppins Bold       (titres, chiffres)
--font-body          Inter              (texte courant)
--font-editorial     Fraunces Italic    (un mot par titre max)
```

## Exemples canoniques

**User :** "Génère-moi une slide stats CACD2 sur 70 collab, 45 projets, 6 ans."

→ Tu utilises `slide-stats.html`, tu mets 3 stats avec ces valeurs. Titre potentiel : "Six ans d'expertise *digital* au service du groupe." (Fraunces italic sur "digital"). Sidebar label = "LE GROUPE". Page = "02 / N".

---

**User :** "Fais-moi une slide titre 'Notre approche' chapitre 03."

→ Tu utilises `slide-title.html`, fond vert sapin (`--brand-green-deep`), "03" en outline géant vert vif, "Notre approche." en blanc bold. Sidebar label vertical = "APPROCHE". Page = "03 / N".

---

**User :** "Rédige une fiche produit pour notre solution AgilAuto en 4 sections."

→ Tu structures : (1) section cover `slide-section.html`, (2) section stats `slide-stats.html`, (3) section content avec mockup `slide-content.html`, (4) citation client `slide-quote.html`. Le tout en français FR, ton sobre, vocabulaire CACD2.

## Que faire si la demande sort du scope

- **Demande de logo** : ne pas générer, pointer vers `Design System/Logo_CACD2_noir_couleur_2_lignes.png`
- **Demande de scène 3D Spline** : pointer vers `../assets/README.md` (intégration via iframe)
- **Demande hors design system** (ex: gros slide d'animation complexe) : prévenir l'humain, proposer une version simplifiée respectueuse du DS

## Sources canoniques

Toutes les informations à utiliser sont dans `content-rules.md` (chiffres, piliers, expertises, marqueurs, cas clients connus). Ne jamais inventer de chiffres ou de projets — si une info manque, demander ou laisser un placeholder explicite `[à confirmer]`.
