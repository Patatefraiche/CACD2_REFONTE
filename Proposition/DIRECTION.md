# CACD2 — Direction artistique de transition

*Document de référence interne · juin 2026 · Proposition de transition pré-fusion plateforme Data*

## En une phrase

**Editorial premium tech avec accents 3D matériel.** On conserve l'identité CACD2 (vert sapin, sidebar éditoriale, chiffres en gros) et on injecte du caractère par du 3D matière, une typo éditoriale mêlant Poppins et Fraunces, et des tuiles modulaires.

## Pourquoi cette direction

CACD2 produit aujourd'hui slides, propals et fiches produits via Claude/Stitch/PowerPoint sans direction unifiée. Résultat : on tombe dans les clichés génériques de l'IA (fusées, nuages, em-dashes, points verts). Cette direction sert de **rampe de transition** jusqu'à la refonte d'identité post-fusion Data AI / Data Lab.

## Palette

### Light mode — primaire

| Token | Code | Usage |
|---|---|---|
| `surface-mint` | `#E8F5EC` | Fond doux signature CACD2 |
| `surface-base` | `#FFFFFF` | Sections de respiration |
| `brand-green` | `#01A04C` | Couleur du logo, accents identitaires |
| `brand-green-deep` | `#0E2E22` | Titres puissants, slides de transition |
| `accent-teal` | `#2EB5A8` | Accent contemporain ponctuel |
| `ink-strong` | `#0A0F0D` | Texte primaire |

### Dark mode — transition

Inversion gérée via `[data-theme="dark"]`. Fond `#0A0F0D` + tonal `#0E2E22`, accent vert vif `#2EE07A` pour le contraste.

### Iridescence (usage strict)

Gradient `#FF7A45 → #E5478B → #7950F2 → #2389E5` — **uniquement** pour le visuel hero 3D. Strictement banni partout ailleurs (boutons, fonds, icônes, gradients de cards).

## Typographie

| Usage | Famille | Pourquoi |
|---|---|---|
| Display | **Poppins Bold** | Conservé du DS existant — solide, géométrique, lisible |
| Body | **Inter** | Substitut moderne à Roboto, optimisé écran |
| Éditorial | **Fraunces Italic** | Signature "Magnetism" — un mot italique dans un titre = caractère |

**Règle d'or :** Fraunces s'utilise pour **un seul mot** dans un titre, ou pour les points finaux dramatiques (`Attraction.`). Jamais pour des paragraphes entiers.

**Exemples :**
- "Le partenaire *digital* du groupe Crédit Agricole."
- "Fabricants et opérateurs de solutions *IT*."
- "Une question, un *besoin* ?"

## Layout & composants

- **Sidebar éditoriale verticale** : label section uppercase (tracking 0.14em) + page indicator (`08/14`) sur les sections importantes
- **Tuiles modulaires** : radius 16-24px, fond `mint` ou `base`, ombre douce
- **Chiffres outline géants** : element graphique de section divider (réf. "04 Expertises")
- **Navigation horizontale type Notion** : pas de menus déroulants complexes

## Iconographie — deux registres distincts

1. **3D matériel** (Spline, photoréaliste) → **identité, hero, branding**
2. **Line-art vert minimaliste** (DS existant) → **fonctionnel, content, fiches**

Ne jamais mélanger les deux dans un même bloc visuel.

## Anti-clichés (interdits stricts)

- Pas de fusées, nuages IA, cerveaux, robots, ampoules
- Pas d'em-dashes (—) injectés dans le copy sans raison
- Pas de point vert "● Generated by Claude" en fin de slide
- Pas de gradients pastels génériques (blobs purple/pink/blue)
- Pas de fond noir comme couleur primaire d'une slide (sauf citation, transition)
- Pas de stock imagery, illustrations Storyset/undraw
- Pas d'emojis dans les titres
- Pas de formules creuses : "transform your business with AI", "next-gen platform", "disrupt the industry"

## Ton éditorial

- **Direct, sobre, factuel.** Comme cacd2.fr : "Le partenaire de la transformation numérique de Crédit Agricole S.A."
- **Pas de superlatifs creux** : éviter "leader", "incontournable", "expert n°1"
- **Chiffres concrets** quand ils existent (70 collaborateurs, 45 projets, 23+ entités)
- **Vocabulaire métier précis** : digital factory, IT as a Service, plateforme data, opérateur de solutions
- **Français FR** : pas d'anglicismes inutiles ("delivery" → "livraison"), sauf termes établis ("design system", "product")

## Évolutions prévues (hors-scope de cette transition)

- Intégration scène Spline 3D réelle (en attente du lien public)
- Animations scroll-driven et micro-interactions
- Templates additionnels : article blog, fiche produit, propal multi-page
- Refonte post-fusion Data AI / Data Lab → ce DS sert de base candidate
